# コードレビュースキル

現在の変更内容をレビューし、フィードバックを提供する。

## 引数

- 引数なし → ステージング済み・未ステージングの全変更をレビュー
- `pr #番号` → 指定PRの変更をレビュー
- `commit HEAD~N..HEAD` → 直近Nコミットの変更をレビュー

## レビュー手順

1. 変更差分を取得する
   - ローカル変更: `git diff` と `git diff --staged`
   - PR: `gh pr diff {番号}`
2. 変更されたファイルの元のコードも読み、コンテキストを把握する
3. 以下の観点でレビューする

### レビュー観点

| 観点 | チェック内容 |
|------|------------|
| 正確性 | ロジックの誤り、エッジケースの考慮漏れ |
| セキュリティ | 機密情報の露出、インジェクション、OWASP Top 10 |
| パフォーマンス | N+1クエリ、不要なループ、メモリリーク |
| 可読性 | 命名、関数の責務分離、複雑度 |
| テスト | テストの有無、カバレッジの十分さ |

4. レビュー結果を以下の形式で報告する

### 報告形式

```
## レビュー結果

### 問題あり（要修正）
- [ファイル:行] 内容

### 提案（任意）
- [ファイル:行] 内容

### 良い点
- 内容
```

## 注意事項

- 重大な問題がなければ「問題なし」と明確に伝える
- 修正提案は具体的なコード例を含める
